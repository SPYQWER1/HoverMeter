Task 12: Refresh After Save Verification
=========================================
Date: 2025-06-23

1. Dynamic refresh interval (no longer hardcoded)
-------------------------------------------------
src/App.tsx:
  const [refreshInterval, setRefreshInterval] = useState<number>(
    DEFAULT_REFRESH_INTERVAL,  // 5
  );
  const { ..., refresh } = useUsageData(refreshInterval);

Previously: useUsageData(5) — hardcoded 5 minutes.
Now: useUsageData(refreshInterval) — driven by saved settings.

On mount, refreshInterval is loaded from invoke("load_settings"):
  if (typeof settings.refresh_interval === "number" &&
      Number.isFinite(settings.refresh_interval)) {
    setRefreshInterval(settings.refresh_interval);
  }

useUsageData hook (src/hooks/useUsageData.ts) useEffect:
  useEffect(() => {
    const ms = Math.max(30_000, refreshIntervalMinutes * 60_000);
    const id = setInterval(() => { fetchData(false); }, ms);
    return () => clearInterval(id);
  }, [refreshIntervalMinutes, fetchData]);

When refreshInterval changes, the old interval is cleared and a new
one is set. The interval is floored at 30 seconds.

2. Save triggers data refresh
------------------------------
src/App.tsx handleSaveSettings:
  const handleSaveSettings = async (settings: AppSettings) => {
    try {
      await invoke("save_credentials", { ... });
      await invoke("save_settings", {
        refreshInterval: settings.refresh_interval,
        opacity: settings.opacity,
      });
      setOpacity(settings.opacity);
      setRefreshInterval(settings.refresh_interval);
      setShowSettings(false);
      refresh();  // <-- triggers immediate data refresh
    } catch (err: unknown) {
      console.error("failed to save settings:", err);
    }
  };

After successful save:
  1. setRefreshInterval() updates the hook's interval (useEffect re-runs)
  2. setShowSettings(false) closes the settings panel
  3. refresh() calls fetchData(false) in useUsageData

The refresh() callback re-loads credentials via invoke("load_credentials")
inside fetchData, so newly saved DeepSeek API key is picked up immediately.

3. Refresh flow in useUsageData
-------------------------------
src/hooks/useUsageData.ts fetchData(false):
  - On refresh (not initial), if hasDeepSeekKey.current is true:
    - Re-loads credentials via invoke("load_credentials")
    - If deepseek_key exists, fetches balance with new key
    - If deepseek_key is missing/empty, sets hasDeepSeekKey.current = false
  - Always fetches volcano usage (arkcli handles auth independently)

This means: after saving new credentials, the refresh() call will
re-read them from keyring and use the new API keys for the next
DeepSeek balance fetch.

4. State update chain
---------------------
save_credentials (keyring) + save_settings (JSON file)
  -> setOpacity (updates CSS --widget-opacity variable)
  -> setRefreshInterval (updates useUsageData auto-refresh interval)
  -> setShowSettings(false) (closes panel)
  -> refresh() (immediate data fetch with new credentials)

5. Error handling
------------------
If save_credentials or save_settings fails (throws), the catch block
logs the error and the settings panel stays open (setShowSettings(false)
is not reached). The user can retry. No partial state update occurs
because setOpacity/setRefreshInterval/setShowSettings/refresh are all
after both await calls succeed.
