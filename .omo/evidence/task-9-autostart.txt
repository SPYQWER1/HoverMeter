Task 9: Autostart + Window-State + Positioner Plugin Evidence
=============================================================
Date: 2025-06-23

## Autostart Plugin (tauri-plugin-autostart)
- Initialized in lib.rs run() function:
  tauri_plugin_autostart::init(MacosLauncher::LaunchAgent, None)
- Cargo.toml dependency: tauri-plugin-autostart = "2"
- Capabilities: "autostart:default" in capabilities/default.json
- The plugin provides enable/disable/isEnabled APIs to the frontend
  via Tauri invoke commands (e.g., @tauri-apps/plugin-autostart)
- No additional Rust-side configuration needed; frontend (Task 12) can
  call enable()/disable() to toggle开机自启

## Window-State Plugin (tauri-plugin-window-state)
- Initialized in lib.rs run() function:
  tauri_plugin_window_state::Builder::default().build()
- Cargo.toml dependency: tauri-plugin-window-state = "2"
- Capabilities: "window-state:default" in capabilities/default.json
- Automatically saves and restores window position, size, and state
  (maximized, minimized, fullscreen) across app restarts
- Uses Builder pattern (not plain init()) for potential future configuration

## Positioner Plugin (tauri-plugin-positioner)
- Initialized in lib.rs run() function:
  tauri_plugin_positioner::init()
- Cargo.toml dependency: tauri-plugin-positioner = "2"
- Capabilities: "positioner:default" in capabilities/default.json
- Provides tray-relative window positioning (e.g., move_window_to_tray)
- Can be used to position the widget near the tray icon on click

## Notes
- All three plugins were already registered in lib.rs from Task 1 scaffold.
- Task 9 preserves all existing plugin registrations (MUST NOT remove).
- The autostart plugin's MacosLauncher::LaunchAgent is the recommended
  launcher type for macOS; on Linux it uses XDG autostart desktop entries.
- The None argument for autostart means no additional launch arguments
  are passed when the app starts at login.
