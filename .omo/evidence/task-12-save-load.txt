Task 12: Save/Load Credentials and Settings Verification
========================================================
Date: 2025-06-23

1. First-launch detection (no credentials -> auto-open settings)
-----------------------------------------------------------------
src/App.tsx mount useEffect:
  invoke<StoredCredentials | null>("load_credentials")
    .then((creds) => {
      if (!creds) {
        setShowSettings(true);
      }
    })

load_credentials Tauri command (storage.rs) returns Option<Credentials>.
When keyring has no saved credentials, it returns None (null in JS),
which triggers setShowSettings(true) — the settings panel auto-opens
on first launch.

2. Pre-fill form on open
-------------------------
src/App.tsx — useEffect watching [showSettings]:
  useEffect(() => {
    if (!showSettings) return;
    Promise.all([
      invoke<StoredCredentials | null>("load_credentials"),
      invoke<StoredSettings>("load_settings"),
    ])
      .then(([creds, settings]) => {
        setInitialSettings({
          volcano_access_key: creds?.volcano_ak ?? "",
          volcano_secret_key: creds?.volcano_sk ?? "",
          deepseek_api_key: creds?.deepseek_key ?? "",
          refresh_interval: settings.refresh_interval,
          opacity: settings.opacity,
        });
      })
      .catch(...);
  }, [showSettings]);

When the settings panel opens (showSettings becomes true), both
credentials and settings are loaded in parallel and passed to the
Settings component as initialSettings prop.

src/Settings.tsx — useEffect syncs initialSettings to local state:
  useEffect(() => {
    if (!isOpen || !initialSettings) return;
    setVolcanoAccessKey(initialSettings.volcano_access_key);
    setVolcanoSecretKey(initialSettings.volcano_secret_key);
    setDeepseekApiKey(initialSettings.deepseek_api_key);
    setRefreshInterval(initialSettings.refresh_interval);
    setOpacity(initialSettings.opacity);
  }, [isOpen, initialSettings]);

All credential inputs remain type="password" (unchanged from Task 8).

3. Save button persists via Tauri commands
------------------------------------------
src/App.tsx handleSaveSettings (async):
  await invoke("save_credentials", {
    volcanoAccessKey: settings.volcano_access_key,
    volcanoSecretKey: settings.volcano_secret_key,
    deepseekApiKey: settings.deepseek_api_key,
  });
  await invoke("save_settings", {
    refreshInterval: settings.refresh_interval,
    opacity: settings.opacity,
  });

Tauri v2 snake_case -> camelCase argument mapping:
  Rust: volcano_access_key -> JS: volcanoAccessKey
  Rust: volcano_secret_key -> JS: volcanoSecretKey
  Rust: deepseek_api_key  -> JS: deepseekApiKey
  Rust: refresh_interval   -> JS: refreshInterval
  Rust: opacity            -> JS: opacity

save_credentials (storage.rs) stores all three keys in OS keyring
via keyring::Entry::set_password().
save_settings (storage.rs) writes { refresh_interval, opacity } as
JSON to app_data_dir/settings.json.
