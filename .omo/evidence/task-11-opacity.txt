Task 11: Opacity Control Verification
=======================================
Date: 2025-06-23

1. CSS variable for opacity
----------------------------
src/styles.css :root:
  --bg-widget: rgba(20, 20, 30, var(--widget-opacity, 0.85));

The --widget-opacity custom property defaults to 0.85 when not set.
When set via JS (on document.documentElement), it overrides the default.
CSS custom property substitution is recursive and resolved at the point
of use, so --widget-opacity set on :root is available when --bg-widget
is consumed by .widget { background: var(--bg-widget); }.

2. Load opacity from settings on mount
---------------------------------------
src/App.tsx mount useEffect:
  invoke<StoredSettings>("load_settings")
    .then((settings) => {
      if (typeof settings.opacity === "number" && Number.isFinite(settings.opacity)) {
        setOpacity(settings.opacity);
      }
    })

load_settings Tauri command (storage.rs) returns Settings { refresh_interval, opacity }.
The command always succeeds (returns defaults if file missing/unreadable).

3. Apply opacity to UI
-----------------------
src/App.tsx opacity useEffect:
  useEffect(() => {
    document.documentElement.style.setProperty(
      "--widget-opacity",
      String(opacity),
    );
  }, [opacity]);

This runs on mount (applying default 0.85) and whenever opacity changes,
providing real-time updates.

4. Real-time opacity preview (live slider)
-------------------------------------------
src/Settings.tsx:
  interface SettingsProps {
    ...
    onOpacityChange?: (opacity: number) => void;
  }

  const handleOpacityChange = (e: ChangeEvent<HTMLInputElement>) => {
    const value = e.target.valueAsNumber;
    setOpacity(value);
    onOpacityChange?.(value);
  };

src/App.tsx passes onOpacityChange={handleOpacityChange} to Settings:
  const handleOpacityChange = (value: number) => {
    setOpacity(value);
  };

When the user drags the opacity slider in Settings, each onChange event
fires handleOpacityChange which calls onOpacityChange, which updates
the opacity state in App, which triggers the useEffect to update the
CSS variable, which immediately changes the widget background opacity.
This provides live preview WITHOUT needing to save first.

5. On save
----------
src/App.tsx handleSaveSettings:
  const handleSaveSettings = (settings: AppSettings) => {
    setOpacity(settings.opacity);
    setShowSettings(false);
  };

Ensures the final saved opacity value is applied (in case it differs
from the last live-preview value, though they should match).
Task 12 will add invoke("save_settings", ...) here for persistence.

6. Opacity range
-----------------
Settings.tsx slider: min=0.5, max=1.0, step=0.05
Range is 0.5 to 1.0 (50% to 100% opacity), matching the plan spec.
