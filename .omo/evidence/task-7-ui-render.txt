Capability change rationale:
  core:window:default (included in core:default) does NOT grant allow-show,
  allow-hide, or allow-start-dragging (verified via src-tauri/gen/schemas/
  acl-manifests.json). These were added explicitly so the required
  show()-after-mount, hide()-to-tray, and drag-region behaviors work at
  runtime. This is capability config, not backend Rust code.

Settings overlay placement note:
  <Settings> is rendered as a SIBLING of .widget (under #root), NOT inside
  .widget. Reason: .widget has backdrop-filter, which makes it the containing
  block for position:fixed descendants; combined with .widget's overflow:hidden
  that would clip/reparent the overlay. As a sibling under #root (no
  transform/filter), the fixed overlay correctly spans the 320x180 viewport.
